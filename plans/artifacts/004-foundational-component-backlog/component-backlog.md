# Foundational Component Backlog

Generated by Plan 004 on 2026-06-24 from the pinned local Base UI and shadcn
submodules. Button is intentionally excluded because `base-ui/button` and
`shadcn/button` are already implemented and should be used as precedent.

## How to Use This Queue

- Run `add-registry-component` against one row at a time, or one paired row at
  a time when the row includes both a Base UI and shadcn URL.
- After the generic resolver work in Plan 004 lands, the rows marked
  `Parallel-safe starter` can be dispatched concurrently in separate worktrees.
- For paired rows, generate the Base UI primitive and shadcn wrapper in the
  same dossier so dependencies stay explicit.
- For shadcn-only rows, keep the item in the `shadcn/` namespace and record
  whether it depends on `utils/cn`, theme tokens, or another local component.

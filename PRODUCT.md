# Product

## Register

product

## Platform

web

## Users

Existing Foldkit developers adding UI components. They use the documentation while building Foldkit applications and need to find an appropriate component, understand how it behaves, and install it with confidence.

## Product Purpose

Foldkit CN helps Foldkit developers quickly discover, evaluate, preview, and install suitable UI components. Success means developers can move from a UI need to a working, idiomatic Foldkit component without uncertainty about availability, behavior, installation, or provenance.

## Positioning

The shadcn component ecosystem, ported idiomatically to Foldkit.

## Brand Personality

Precise, pragmatic, and Foldkit-native. The documentation should feel familiar to developers who know shadcn while placing Foldkit's architecture, conventions, examples, and identity at the center of the experience.

## Anti-references

Avoid a superficial shadcn reskin that merely renames the source material or obscures how the component has been translated into Foldkit. Familiar documentation affordances should support comprehension, not displace the Foldkit-specific model and differences.

## Design Principles

- Preserve familiar component-documentation workflows so developers can orient themselves immediately.
- Make the Foldkit translation explicit through architecture, conventions, examples, and provenance.
- Show installability and lifecycle status honestly so unavailable work never looks ready to use.
- Keep discovery, evaluation, preview, and installation connected as one efficient workflow.
- Practice Foldkit's own standards in the documentation site's behavior and implementation.

## Accessibility & Inclusion

Meet WCAG 2.2 AA as the baseline. Support complete keyboard operation, clearly visible focus, sufficient text and control contrast, semantic structure, and reduced-motion preferences.
